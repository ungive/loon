◆ fail_on_too_many_requests

bool loon::ClientOptions::fail_on_too_many_requests { false }

If set to true, this client fails on too many requests.

How many requests exactly consitute as "too many requests" is determined by the value for max_requests_per_second, which must be set if this flag is true.

On failure, the client stops with an error and must be started again manually to continue.

◆ max_requests_per_second

std::optional<double> loon::ClientOptions::max_requests_per_second {}

The maximum number of requests to handle per second.

If this number is exceeded, the client restarts. Any registered content will be unregistered and must be registered again. Unregistered callbacks will be called on each piece of content.

If fail_on_too_many_requests is set, the client fails instead.

Maximum total upload speed for the client in bytes per second.

Limits the upload speed for all requests that are served by the client. If multiple requests are served simultaneously, this limit is evenly split across every response on a best-effort basis.

◆ min_cache_duration

std::optional<std::chrono::seconds> loon::ClientOptions::min_cache_duration {}

The minimum duration for which responses must be cached by the server, in seconds.

This means that when another request comes in for the same content, within the time period of this duration, the client will fail and stop with an error.

The client also fails if the server does not support caching.
